# 5进程API
## 5.1fork
```
pid= fork()

pid==0 表示 子进程执行
pid >0 父进程执行
   pid 表示的fork 之后 子进程的进程号



note:
   如果该电脑只有一个CPU,那么fork() 子进程被调度和父进程被调度都有可能被先调度

```

## 5.2 wait
```
父进程 调用wait() 等待子进程结束,wait() 才返回
```
## 5.3 exec调用
```
exec 调用 会覆盖当前自己代码端和数据段 并被重新初始化
```
