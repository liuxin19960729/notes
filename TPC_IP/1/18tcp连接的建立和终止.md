# 18 TCP连接的建立和终止
## 18.2.4 连接终止协议
```
终止一个连接需要4次握手 这是由于TCP 半连接造成(halfclose)

FIN-->recv->通知应用层另一端已经发送关闭了发送数据

note: 搜到FIN 只能表示一个方向终止了数据发送但是并不代表TCP 不能数据发送 没有发送FIN 数据的方向仍然能发送数据(有一些应用就是利用办连接来接收数据的)

接收到FIN 应用层会获取到一个文件EOF 字符
```
